나는 "시장 심리 흐름 시각화 대시보드"를 만들고 싶다.

이 프로젝트의 핵심 목표는:
- 특정 종목의 미래 가격을 단정적으로 예측하는 것이 아니다.
- 시장 심리와 관심 흐름을 시각적으로 이해하는 것이다.
- 여러 심리 시그널을 한 화면에서 비교해서 "현재 시장의 심리적 위치"를 파악하는 것이 목적이다.
- 완벽한 바닥/천장을 맞추려는 시스템이 아니다.
- 이미 방향이 바뀐 직후라도 흐름을 남들보다 조금 더 빨리 인지하는 것이 중요하다.
- 후행 데이터도 의미가 있다.
- 검색량과 뉴스량이 이미 증가한 후라도 관심 확산이 아직 진행 중이라면 흐름이 지속될 수 있다.
- 사용자는 여러 시그널을 누적해서 1%라도 더 나은 판단을 하고 싶어 한다.

중요한 관점:
- 시장은 단순히 재무 데이터만으로 움직이지 않는다.
- 대중의 관심, 검색, 뉴스, SNS, 커뮤니티 확산, FOMO가 실제 거래량과 가격에 영향을 준다.
- 이 프로젝트는 "관심의 퍼널"과 "심리 에너지 흐름"을 시각적으로 관찰하는 도구다.

예시:
1. 일부 사람만 먼저 관심
2. 검색량 증가
3. 커뮤니티 확산
4. 뉴스 증가
5. 유튜브/SNS 확산
6. 거래량 증가
7. FOMO
8. 과열 또는 추세 지속

이런 흐름을 관찰하고 싶다.

기술 스택:
- Frontend: Next.js + React + TypeScript
- Styling: Tailwind CSS
- Chart: lightweight-charts 또는 TradingView 스타일 라이브러리 우선
- Backend/API: Next.js API Routes 또는 FastAPI
- Data Storage:
  - 초기: SQLite 또는 JSON 캐시
  - 추후: PostgreSQL/Supabase 확장 가능 구조
- Python 사용 가능
- 데이터 수집은 자동화 가능하게 설계

핵심 MVP 목표:
"한 화면에서 시장 심리 흐름을 시각적으로 비교할 수 있는 대시보드"

3. Google Trends 연동
==================================================

매우 중요:
Google Trends는 절대 검색량이 아니라 "상대적 관심도 지표"라는 점을 반드시 고려해라.

즉:
- 실제 검색 횟수는 알 수 없다.
- 선택된 기간/지역 내 최고 관심도를 100으로 정규화한 데이터다.
- 하지만 이 프로젝트의 목적은 절대 검색량이 아니라:
  - 관심 증가 속도
  - 관심 지속성
  - 관심 확산 타이밍
  - 가격/거래량 대비 관심 변화
를 보는 것이므로 충분히 의미 있다.

구현 방식:
- pytrends 사용
- 비공식 API 기반이라는 점 README에 명시
- 캐싱 시스템 구현
- 매번 수동 확인하지 않도록 자동 수집 구조 설계

반드시 포함:
- 지역 고정 기능
  - KR
  - US
  - JP
- 기간 고정 기능
- 키워드 다중 등록
  예:
  - Tesla
  - TSLA stock
  - 테슬라 주식
- 수집 시점 저장
- 이동평균 계산
- 증가율 계산
- 스파이크 감지

중요:
Google Trends 수치는 재정규화될 수 있으므로:
- 원본 값
- 이동평균
- 증가율
- 기준 기간
을 함께 저장해라.

==================================================

- 검색량 증가율

- 거래량 증가율
- 가격 수익률
- 관심 스파이크
- 검색량 대비 가격 반응
- 뉴스량 대비 거래량 반응

예시 해석:
- 검색량은 급증했는데 가격 반응 약함

- 부정 뉴스인데 가격이 안 떨어짐
- 거래량은 증가했는데 뉴스량은 적음

이런 "비대칭"을 발견할 수 있게 만들어라.
